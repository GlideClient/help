Mod Blacklisting System

25/07/2025
BreadCat

This article refers GlideClient 7.2 (7201) and above.

GlideClient has a blacklisting feature now.

Starting from the release of GlideClient 7.2 we implimented a way for servers to stop you using mods they explicitly dont allow.
GlideClient now checks a online json to see if the server you joined is blocking the mod you are trying to enable.
If a mod you are using is blacklisted then the client will disable it, send a notification and hide it from your mod menu.
Note: Glide may not always blacklist mods so be sure to check the rules of the servers you are playing on as all mods on glide are use at own risk.

For server owners.

If you want to get your server added to our list of servers that have blacklists there is two ways you can do that.
You can make a pull request on our github data repo.
You can contact us on on our Discord Server.
Warning: GlideClient by defualt may have mods that are banable on servers so its best that you do add yourself to the list.

For users.

We understand you may face bugs so we did add an override to the blacklist feature.
Note: Using blacklisted mods on servers that are telling you not to may and will result in bans on those servers. We ask you dont abuse this feature to break rules of servers.
To disable this feature you can add -Dme.eldodebug.soar.glideclient.blacklistchecks=false to your jvm launch arguments for glide.
